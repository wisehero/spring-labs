batch INSERT (IDENTITY 불가)
⚠
Native Bulk가 JdbcTemplate보다 느린 이유:
chunked(500)으로 나눠서 전송하므로 10,000건 기준 20번의 네트워크 왕복이 발생합니다. 반면 JdbcTemplate + rewriteBatchedStatements=true는 MySQL JDBC 드라이버가 자동으로 단일 multi-row INSERT로 변환하여 네트워크 왕복을 최소화합니다. PreparedStatement 재사용까지 더해져 가장 빠른 성능을 보입니다.
🔴
JPA IDENTITY 전략과 배치 INSERT의 치명적 충돌:
IDENTITY 전략은 INSERT 직후 DB에서 생성된 ID를 즉시 조회해야 합니다. 이로 인해 Hibernate가 batch INSERT를 비활성화하고 건별 INSERT를 수행합니다. saveAll()을 사용하더라도 배치의 이점을 전혀 누릴 수 없습니다.

Section 05

추가 최적화 팁

// ❌ IDENTITY 전략 — Hibernate가 배치를 자동 비활성화 @Entity class Book( @Id @GeneratedValue(strategy = GenerationType.IDENTITY) val id: Long? = null ) // INSERT 후 즉시 SELECT LAST_INSERT_ID() → 배치 불가
// ✅ SEQUENCE 전략 — 배치 INSERT 가능 @Entity @SequenceGenerator(name = "book_seq", sequenceName = "book_seq", allocationSize = 50) class Book( @Id @GeneratedValue(strategy = GenerationType.SEQUENCE, generator = "book_seq") val id: Long? = null ) // allocationSize=50 → 50개씩 ID 미리 할당 → 배치 INSERT 정상 동작
-- 대량 삽입 전 보조 인덱스 임시 비활성화 (MyISAM only) ALTER TABLE book DISABLE KEYS; -- 대량 INSERT 수행 INSERT INTO book (title, author, price) VALUES ('책1', '저자1', 15000), ('책2', '저자2', 23000), -- ... N개 rows ... -- 삽입 완료 후 인덱스 재활성화 (일괄 인덱스 빌드) ALTER TABLE book ENABLE KEYS; -- InnoDB는 innodb_autoinc_lock_mode=2 (interleaved) 설정 권장 -- SET foreign_key_checks=0 도 함께 사용 가능
-- 가장 빠른 방법: LOAD DATA INFILE (파일 기반 직접 적재) LOAD DATA INFILE '/var/lib/mysql-files/books.csv' INTO TABLE book FIELDS TERMINATED BY ',' OPTIONALLY ENCLOSED BY '"' LINES TERMINATED BY '\n' IGNORE 1 ROWS (title, author, price); -- Spring에서 사용 시: LOCAL INFILE 활성화 필요 -- allowLoadLocalInfile=true (JDBC URL 파라미터) -- 초당 수십만 건 처리 가능 — 마이그레이션/초기 적재에 적합
